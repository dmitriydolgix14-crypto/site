# Horror Site - Система авторизации и отзывов

Полноценная система авторизации с бэкендом на Node.js + Express + MongoDB.

## 🚀 Требования

- Node.js (v18 или выше)
- MongoDB (локально или MongoDB Atlas)
- npm или yarn

## 📦 Установка

1. **Клонируй репозиторий:**
```bash
git clone https://github.com/dmitriydolgix14-crypto/site.git
cd site
```

2. **Установи зависимости:**
```bash
npm install
```

3. **Настрой MongoDB:**

**Вариант 1: Локальный MongoDB**
```bash
# Установи MongoDB с официального сайта
# Запусти MongoDB сервер
mongod
```

**Вариант 2: MongoDB Atlas (облачный)**
1. Зарегистрируйся на https://www.mongodb.com/cloud/atlas
2. Создай бесплатный кластер
3. Получи строку подключения
4. Обнови `.env` файл

4. **Настрой переменные окружения:**

Создай файл `.env` в корне проекта:
```env
PORT=3000
MONGODB_URI=mongodb://localhost:27017/horror-site
SESSION_SECRET=твой-секретный-ключ-поменяй-это
JWT_SECRET=твой-jwt-секрет-поменяй-это
```

Для MongoDB Atlas:
```env
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/horror-site?retryWrites=true&w=majority
```

## 🎯 Запуск

**Режим разработки:**
```bash
npm run dev
```

**Продакшн режим:**
```bash
npm start
```

Сервер запустится на `http://localhost:3000`

## 📱 Использование

### Страницы:
- `/` - Главная страница
- `/login.html` - Вход в систему
- `/register.html` - Регистрация
- `/profile` - Профиль (только для авторизованных)

### API Endpoints:

**Авторизация:**
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход
- `POST /api/auth/logout` - Выход
- `GET /api/auth/me` - Получить текущего пользователя

**Отзывы:**
- `GET /api/reviews` - Получить все отзывы
- `POST /api/reviews` - Создать отзыв (требуется авторизация)
- `DELETE /api/reviews/:id` - Удалить отзыв (только свой)

## 🧪 Тестирование

1. **Запусти сервер:**
```bash
npm run dev
```

2. **Открой в браузере:**
```
http://localhost:3000
```

3. **Протестируй регистрацию:**
- Перейди на `/login.html`
- Нажми "Зарегистрироваться"
- Заполни форму:
  - Имя пользователя: `testuser`
  - Email: `test@example.com`
  - Отображаемое имя: `Тестовый Пользователь`
  - Пароль: `password123`
- Нажми "Зарегистрироваться"

4. **Протестируй вход:**
- Войди с теми же данными
- Должен произойти автоматический переход на `/profile`

5. **Протестируй отзывы:**
- На странице профиля найди секцию "Отзывы"
- Напиши отзыв и выбери оценку
- Нажми "Отправить отзыв"
- Отзыв должен появиться в списке

6. **Протестируй выход:**
- Нажми кнопку "Выйти"
- Должен произойти переход на `/login.html`

## 🐛 Решение проблем

### MongoDB не подключается:
```bash
# Проверь, запущен ли MongoDB
mongod

# Или проверь строку подключения в .env
```

### Порт занят:
```bash
# Измени PORT в .env файле
PORT=3001
```

### Ошибка при установке зависимостей:
```bash
# Очисти кэш npm и переустанови
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

## 🚀 Деплой на Vercel

Для деплоя на Vercel нужен внешний MongoDB (MongoDB Atlas):

1. Создай аккаунт на MongoDB Atlas
2. Получи строку подключения
3. Добавь переменные окружения в Vercel:
   - `MONGODB_URI` - твоя строка подключения MongoDB
   - `SESSION_SECRET` - случайная строка
   - `JWT_SECRET` - случайная строка

4. Задеплой на Vercel:
```bash
vercel --prod
```
